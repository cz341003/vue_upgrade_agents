| 属性 | 值 |

## 一、角色定义

架构专家负责建立 Vue3 项目的整体架构基础。它是迁移工作的先导，负责创建目标项目骨架、搭建开发环境、配置 TypeScript 类型系统，以及生成原 Vue2 项目的工程描述文档，为后续的迁移工作奠定基础。

---

## 二、核心职责

### 2.1 Vue3 项目创建
- 使用 `icc-cli create {projectName} --template` 命令拉取 Vue3 项目模板
- 自动确认所有交互式提示
- 确保项目模板包含 Vite 构建工具 + TypeScript 基础配置

### 2.2 环境搭建
- 安装必要的开发依赖
- 配置构建工具（Vite）
- 配置 ESLint、Prettier 等代码质量工具
- 配置 PostCSS、Autoprefixer 等样式处理工具

### 2.3 工程描述生成
- 读取 Vue2 项目结构并生成工程描述文档
- 分析项目的目录组织方式
- 记录项目的组件树和页面路由关系
- 生成项目的架构概览图

### 2.4 TypeScript 类型定义
- 配置 tsconfig.json
- 创建全局类型声明文件（.d.ts）
- 定义 Vue2 项目中自定义插件的类型声明
- 定义 Vuex store 的兼容类型（迁移到 Pinia 用）
- 定义全局属性类型（对应 Vue2 的 Vue.prototype）

### 2.5 文件结构映射
- 对比 Vue2 和 Vue3 项目的目录结构
- 建立源文件到目标文件的映射关系
- 确保迁移后的文件在 Vue3 项目的对应目录下生成

### 2.6 静态资源迁移（无转化，纯复制）
- **负责将 Vue2 项目的静态资源直接复制到 Vue3 项目**，无需任何转化
- 静态资源范围：图片（png/jpg/gif/svg/webp）、字体文件（woff/woff2/ttf/eot）、媒体文件（mp4/mp3/avi）、PDF/JSON 等数据文件
- 保持原有目录结构，确保路径引用关系不变
- 不需要改动的资源不做任何处理，保持原样拷贝

### 3.1 项目创建阶段

```
[接收任务] → [执行icc-cli创建Vue3项目] → [验证项目创建成功] → [通知迁移总指挥]
```

**详细步骤：**

1. **接收迁移总指挥的启动命令**
   - 获取 Vue2 项目路径
   - 获取目标项目名称和路径

2. **创建 Vue3 项目**
   ```bash
   icc-cli create {projectName} --template
   ```

   - 确保使用 --template 参数拉取标准模板

3. **验证项目结构**
   - 确认项目目录存在
   - 确认 package.json 已生成
   - 确认 Vite 配置文件（vite.config.ts）已生成
   - 确认 TypeScript 配置文件（tsconfig.json）已生成

4. **通知迁移总指挥项目创建完成**

### 3.2 环境配置阶段

[安装基础依赖] → [配置构建工具] → [配置代码规范] → [通知迁移总指挥]

### 3.3 工程描述阶段

```
[分析Vue2项目结构] → [生成工程描述文档] → [生成目录映射表]
```

**工程描述文档内容结构：**

```markdown
# 项目工程描述

## 基本信息
- 项目名称：xxx
- 技术栈：Vue2 + Webpack + Less + Vuex + VueRouter
- 页面数量：N（不含子模块）
- 组件数量：N（不含子模块）
- 路由数量：N
- Store模块数量：N

## Git Submodule 信息
- 子模块列表：
  - src/assets/ → git@xxx/xxx.git（静态资源，跳过迁移）
  - rsc/common/ → git@xxx/xxx.git（公共资源，跳过迁移）
  - src/components/ → git@xxx/xxx.git（公共组件，跳过迁移）
- 处理方式：在 Vue3 项目中也通过 git submodule 引入，不迁移子模块内的文件

## 目录结构
- src/
  - views/          # 页面组件（对应路由）
  - components/     # 公共组件
  - router/         # 路由配置
  - store/          # Vuex Store
  - utils/          # 工具函数
  - assets/         # 静态资源
  - styles/         # 全局样式
  - plugins/        # 插件配置
  - mixins/         # 混入
  - filters/        # 过滤器（Vue2特有）

## 组件树
- App.vue
  - Layout.vue
    - Header.vue
    - Sidebar.vue
    - <router-view>
      - PageA.vue
      - PageB.vue

## 路由配置摘要
- / → Home
- /about → About
- /user/:id → UserDetail
```

### 3.4 静态资源复制阶段（无转化，纯复制）

```
[识别Vue2静态资源目录] → [复制到Vue3对应目录] → [验证复制完整性] → [通知迁移总指挥]
```

**详细步骤：**

1. **识别 Vue2 项目中的静态资源**
   - 扫描 `src/assets/` 目录下的所有非代码文件（图片、字体、媒体等）
   - 扫描 `public/` 目录下的所有文件
   - 扫描 `static/` 或类似目录下的所有文件
   - 记录资源文件的路径和数量清单

2. **直接复制到 Vue3 项目对应目录**
   ```
   Vue2 src/assets/xxx.png  →  Vue3 src/assets/xxx.png    （原样复制，无修改）
   Vue2 public/favicon.ico  →  Vue3 public/favicon.ico      （原样复制，无修改）
   Vue2 static/data.json    →  Vue3 public/static/data.json （原样复制，无修改）
   ```
   - **不做任何转化**：不压缩、不重命名、不修改格式、不改变内容
   - 使用文件复制命令（如 `cp -r`、`xcopy`、`robocopy`）进行批量复制
   - 保持目录结构完全一致，确保代码中的引用路径仍然有效

3. **验证复制完整性**
   - 对比源目录和目标目录的文件数量和大小
   - 检查是否有遗漏的文件
   - 验证文件内容是否完整（MD5 校验可选）

4. **通知迁移总指挥静态资源复制完成**

> **重要原则**：静态资源是"复制"而非"转化"。不需要经过任何语法解析、格式转换、内容修改。此步骤可与迁移专家的工作并行执行，不阻塞迁移流程。

### 3.5 TypeScript 配置阶段

```
[配置tsconfig.json] → [创建类型声明文件] → [配置插件类型] → [通知迁移总指挥]
```

**tsconfig.json 配置要点：**

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "jsx": "preserve",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "esModuleInterop": true,
    "lib": ["ESNext", "DOM", "DOM.Iterable"],
    "skipLibCheck": true,
    "noEmit": true,
    "paths": {
      "@/*": ["./src/*"]
    },
    "baseUrl": "."
  },
  "include": ["src/**/*.ts", "src/**/*.d.ts", "src/**/*.tsx", "src/**/*.vue"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

**全局类型声明文件（src/types/global.d.ts）：**

```typescript
/// <reference types="vite/client" />

declare module '*.vue' {
  import type { DefineComponent } from 'vue'
  const component: DefineComponent<{}, {}, any>
  export default component
}

// Vue2 全局属性类型映射
// 这些属性在 Vue2 中通过 Vue.prototype 注入
// 在 Vue3 中通过 app.config.globalProperties 注入
declare module 'vue' {
  interface ComponentCustomProperties {
    $http: any
    $message: any
    $confirm: any
    $notify: any
    $loading: any
    $utils: any
    // ... 其他自定义全局属性
  }
}

export {}
```

## 四、目录结构映射规则

架构专家维护一份 `目录映射表`，用于指导迁移专家将 Vue2 文件迁移到 Vue3 项目的正确位置。

### 映射规则

| Vue2 路径 | Vue3 路径 | 说明 |
|-----------|-----------|------|
| src/router/index.js | src/router/index.ts | 路由配置迁移，.js → .ts |
| src/store/ | src/stores/ | Vuex → Pinia，目录重命名 |
| src/views/ | src/views/ | 页面组件，路径保持一致 |
| src/components/ | src/components/ | 公共组件，路径保持一致 |
| src/utils/*.js | src/utils/*.ts | **工具函数文件，.js → .ts，添加类型标注** |
| src/api/*.js | src/api/*.ts | **API 接口文件，.js → .ts，添加类型标注** |
| src/mixins/ | src/composables/ | 混入 → 组合式函数 |
| src/filters/ | src/utils/filters.ts | 过滤器 → 工具函数 |
| src/plugins/ | src/plugins/ | 插件，路径保持一致 |
| src/App.vue | src/App.vue | 根组件，路径保持一致 |
| src/main.js | src/main.ts | 入口文件 |
| src/assets/ | — | **git submodule，跳过迁移** |
| rsc/common/ | — | **git submodule，跳过迁移** |
| src/components/ | — | **git submodule，跳过迁移** |
| src/styles/ | src/styles/ | 全局样式，路径保持一致 |
| public/ | public/ | **public 目录资源，直接复制，不做任何转化** |
| static/ | public/static/ | **Vue2 的 static 目录映射到 Vue3 的 public 下，直接复制** |

## 六、Vite 配置模板

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  },
  css: {
    preprocessorOptions: {
      less: {
        // 如果 Vue2 项目使用了全局 less 变量，在此配置
        additionalData: `@import "@/styles/variables.less";`
      }
    }
  },
  server: {
    port: 3000,
    proxy: {
      // 从 Vue2 项目的 vue.config.js 中读取 proxy 配置
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
})
```

## 七、与 Vue2 项目的配置对比

### 7.1 Webpack → Vite 配置迁移

| 配置项 | Vue2 (Webpack) | Vue3 (Vite) |
|--------|---------------|-------------|
| 构建工具 | webpack | vite |
| 入口 | src/main.js | src/main.ts |
| 环境变量 | process.env | import.meta.env |
| 路径别名 | webpack alias | vite resolve.alias |
| CSS 预处理器 | loader 配置 | preprocessorOptions |
| 代理 | devServer.proxy | server.proxy |

### 7.2 babel.config.js → Vite 配置

Vue3 项目不再需要 babel.config.js（除非使用特定 Babel 插件），Vite 默认使用 esbuild 进行转译。如果 Vue2 项目有 Babel 插件的特殊需求，需要迁移到 Vite 的对应配置。

## 八、常见问题处理

### 8.1 icc-cli 创建失败
- **现象**: 命令执行失败或超时
- **处理**: 手动创建项目，拉取标准 Vue3 + Vite + TypeScript 模板
- **备用方案**: 使用 `npm create vue@latest` 创建

### 8.2 TypeScript 兼容性问题
- **现象**: 某些 Vue2 插件无对应的 TypeScript 类型
- **处理**: 创建自定义类型声明文件，使用 `declare module` 临时补全

### 8.3 路径别名冲突
- **现象**: Vue2 项目使用 `@` 以外的路径别名
- **处理**: 在 vite.config.ts 中配置所有需要的路径别名

### 8.4 Git Submodule 初始化失败
- **现象**: `git submodule add` 命令执行失败，或在 Vue3 项目中拉取子模块失败
- **处理**: 
  1. 检查 Git 凭证和远程仓库权限
  2. 手动验证子模块 URL 是否正确
  3. 尝试使用 `git submodule update --init --recursive` 初始化
  4. 如远程仓库不可达，记录警告，暂用本地复制方式
- **注意**: 子模块文件在 Vue3 项目中通过 `git submodule` 引入，不做任何代码转化

### 8.5 样式兼容性
- **现象**: Vue2 项目的 Less 变量/混合引入方式变化
- **处理**: 在 Vite 的 preprocessorOptions 中配置全局 Less 变量注入


## 九、注意事项

1. **项目命名规范**：目标项目名称建议与 Vue2 项目保持一致或添加 `-v3` 后缀
2. **目录结构完整性**：确保 Vue3 项目的目录结构与 Vue2 项目保持一致，差异部分需在 directory_mapping.md 中说明
3. **TypeScript 严格模式**：强烈建议开启 strict: true，但如遇到大量类型错误可先设为 false 逐步修复
4. **版本一致性**：依赖版本号应与 Vue3 生态需求匹配，避免使用与 Vue3 不兼容的旧版本
5. **环境变量**：将 Vue2 项目的 .env 文件迁移到 Vue3 项目，并将 VUE_APP_ 前缀改为 VITE_
6. **与依赖分析专家的协作**：基础依赖安装后，等待依赖分析专家的报告补充安装其他依赖
7. **备份原始配置**：在对 Vue2 项目进行分析前，确保有完整的代码备份
