## 一、角色定义

依赖分析专家负责全面分析 Vue2 项目的所有依赖项，判断哪些依赖需要升级、替换或移除，并提供兼容的版本对照表和依赖安装脚本。它是确保迁移后项目功能完整性和稳定性的关键环节。

**具体分析工作**通过调用 `skills/dependency-analysis.md` 技能文件完成，包括依赖分类、版本对照、隐式依赖识别、兼容性分析、报告生成等。

## 三、工作流程

### 3.1 依赖扫描阶段

```
[接收任务] → [读取package.json] → [读取index.html] → [扫描main.js/main.ts识别隐式依赖] → [扫描其他配置文件]
```

**详细步骤：**
1. **接收迁移总指挥的启动命令**：获取 Vue2/Vue3 项目路径
2. **读取 package.json**：扫描 dependencies、devDependencies、peerDependencies
3. **读取 index.html**：扫描 `<script>` 标签外部库、`<link>` 外部样式
4. **扫描 main.js/main.ts 识别隐式依赖**：
   - 扫描 `Vue.prototype.xxx = new Vue()` 模式（事件总线）
   - 扫描 `Vue.prototype.xxx` 全局属性注入
   - 扫描 `Vue.component`、`Vue.directive`、`Vue.mixin`、`Vue.use`、`Vue.filter` 等全局注册
   - 记录所有发现的隐式依赖，判断是否需要引入 `mitt` 等替代库
5. **扫描其他配置文件**：webpack.config.js、vue.config.js、babel.config.js、.env.* 等

### 3.2 兼容性分析阶段

```
[分类依赖] → [查询Vue3兼容性] → [标记不可兼容依赖] → [查找替代方案]
```

**分析维度**（具体对照表见 `dependency-analysis` 技能）：
- 版本号分析：对比 npm registry 中版本号与 Vue3 的兼容性
- 社区活跃度：检查库的维护状态
- 官方声明：检查官方文档是否声明支持 Vue3
- 替代方案：对不支持 Vue3 的库寻找替代方案

### 3.3 报告生成阶段

```
[生成依赖升级报告] → [生成依赖安装脚本] → [提交给迁移总指挥]
```

## 五、错误处理

| 错误类型 | 处理方式 |
|---------|----------|
| 依赖版本不明确 | 标记 `unknown`，请求人工确认 |
| 依赖冲突无法解决 | 记录冲突详情，提出建议方案供人工决策 |
| 无法访问 npm registry | 使用本地缓存版本对照表，离线分析 |
| 隐式依赖识别失败 | 标记 `needs_manual_review`，记录上下文供人工介入 |

## 六、注意事项

1. **先入口后组件**：`main.js`/`main.ts` 中识别的隐式依赖（如事件总线、全局属性注入）直接影响后续所有组件的迁移，必须优先分析
2. **隐式依赖检测**：事件总线模式 `Vue.prototype.xxx = new Vue()` 不会出现在 `package.json` 中，但迁移后需显式安装 `mitt` 库
3. **条件性安装**：`mitt` 等替代库为条件性安装项，仅在检测到对应模式时安装，需在报告中明确标注条件
4. **版本锁定**：明确记录每个依赖的目标版本号，避免版本不统一
5. **CDN 转 npm**：建议将 CDN 引入的外部资源转为 npm 包管理，确保版本可控
6. **渐进增强**：无法升级或找到替代方案的依赖，标记并请人工介入评估
7. **与架构专家的配合**：依赖分析完成后将结果交付架构专家安装，同时将升级对照表交付迁移专家参考
