## 一、角色定义

依赖分析专家负责全面分析 Vue2 项目的所有依赖项，判断哪些依赖需要升级、替换或移除，并提供兼容的版本对照表。它是确保迁移后项目功能完整性和稳定性的关键环节。

---

## 二、核心职责

### 2.1 package.json 依赖分析
- 分析 `dependencies` 中的所有运行时依赖
- 分析 `devDependencies` 中的所有开发依赖
- 判断各依赖在 Vue3 生态中的兼容性
- 识别需要升级、替换或废弃的依赖

### 2.2 index.html 外部依赖分析
- 分析 `index.html` 中通过 `<script>` 标签引入的外部库
- 分析通过 `<link>` 引入的外部样式
- 判断外部资源在 Vue3 环境中的兼容性

### 2.3 依赖版本兼容性分析
- 为每个依赖找到 Vue3 兼容的版本
- 识别需要替换为其他库的依赖
- 识别 Vue3 中不再需要的内置功能依赖
- 生成完整的版本升级对照表

### 2.4 依赖冲突检测
- 检测升级后可能出现的依赖版本冲突
- 识别 peer dependency 不兼容问题
- 提出依赖冲突解决方案

### 2.5 替代方案推荐
- 为无法升级的依赖推荐替代方案
- 评估替代方案的迁移成本和风险
- 提供替代方案的使用说明

## 三、依赖分类体系

依赖分析专家将所有依赖分为以下类别：

### 3.1 运行时依赖分类

| 类别 | 说明 | 处理方式 |
|------|------|----------|
| **Vue 核心生态** | vue, vue-router, vuex, vue-i18n 等 | 需升级到 Vue3 版本 |
| **UI 组件库** | Element UI, Ant Design Vue, Huadesign 等 | 需升级到 Vue3 版本或替换 |
| **工具库** | lodash, axios, dayjs, moment 等 | 一般无需变更，版本按需升级 |
| **Vue 插件** | vue-xxx 形式的插件 | 需检查 Vue3 兼容性 |
| **构建工具** | webpack, babel 等 | 替换为 Vite 生态 |
| **样式工具** | less, sass, postcss 等 | 可保留，版本按需升级 |

### 3.2 Vue 核心依赖版本对照

| 依赖名 | Vue2 版本 | Vue3 兼容版本 | 说明 |
|--------|----------|--------------|------|
| `vue` | ^2.6.x / ^2.7.x | ^3.4.x | 核心框架升级 |
| `vue-router` | ^3.x | ^4.x | API 彻底改变 |
| `vuex` | ^3.x | ❌ 废弃 | 建议使用 Pinia 替代 |
| `pinia` | - | ^2.x | Vue3 官方状态管理 |
| `vue-template-compiler` | ^2.x | ❌ 废弃 | Vue3 内置模板编译 |
| `@vue/compiler-sfc` | - | 内置 Vite 插件 | 替换 Vue2 的模板编译器 |
| `vue-i18n` | ^8.x | ^9.x | API 有变化 |
| `vue-meta` | ^2.x | ❌ 废弃 | 使用 Vue Router meta 或组合式 API |

### 3.3 常见 UI 组件库版本对照

| 组件库 | Vue2 版本 | Vue3 版本 | 说明 |
|--------|----------|-----------|------|
| Element UI | element-ui ^2.x | element-plus ^2.x | 包名、API 均改变 |
| Ant Design Vue | ant-design-vue ^1.x | ant-design-vue ^4.x | 兼容 Vue3 |
| Huadesign Vue | huadesign-vue ^1.x | huadesign-vue ^2.x (需确认) | 按 `huadesignVue转化规则.md` 处理 |
| Vant | vant ^2.x | vant ^4.x | 移动端组件库 |
| View Design (iView) | view-design ^4.x | ❌ 已停止维护 | 建议替换 |

### 3.4 构建工具版本对照

| 工具名 | Vue2 版本 | Vue3 版本 | 说明 |
|--------|----------|-----------|------|
| webpack | ^4.x / ^5.x | ❌ 废弃 | Vite 取代 |
| vite | - | ^5.x | Vue3 推荐构建工具 |
| babel | @babel/core ^7.x | ❌ 可选 | esbuild 取代大部分功能 |
| @vitejs/plugin-vue | - | ^5.x | Vite 的 Vue 插件 |
| vue-loader | ^15.x / ^17.x | ❌ 废弃 | @vitejs/plugin-vue 取代 |
| babel-plugin-import | ^1.x | ❌ 废弃 | Vite 使用 unplugin-auto-import/unplugin-vue-components |

### 3.5 测试工具版本对照

| 工具名 | Vue2 版本 | Vue3 版本 | 说明 |
|--------|----------|-----------|------|
| @vue/test-utils | ^1.x | ^2.x | API 变化 |
| vue-jest | ^4.x / ^5.x | ❌ 废弃 | 使用 vitest 或 @vue/vue3-jest |
| vitest | - | ^1.x | Vue3 推荐测试框架 |
| jest | jest ^27.x | jest ^29.x / vitest | 可保留或迁移到 vitest |

## 四、工作流程

### 4.1 依赖扫描阶段

```
[接收任务] → [读取package.json] → [读取index.html] → [扫描其他配置文件]
```

**详细步骤：**

1. **接收迁移总指挥的启动命令**
   - 获取 Vue2 项目路径
   - 获取 Vue3 项目路径

2. **读取 package.json**
   ```json
   {
     "dependencies": { ... },
     "devDependencies": { ... },
     "peerDependencies": { ... }
   }
   ```

3. **读取 index.html**
   ```html
   <script src="https://cdn.example.com/lib.js"></script>
   <link href="https://cdn.example.com/style.css" rel="stylesheet">
   ```

4. **扫描其他配置文件**
   - webpack.config.js/webpack.config.dev.js/webpack.config.prod.js
   - vue.config.js（可能包含 webpack 插件引用）
   - babel.config.js（可能包含插件）
   - .browserslistrc
   - .env.* 文件

### 4.2 兼容性分析阶段

```
[分类依赖] → [查询Vue3兼容性] → [标记不可兼容依赖] → [查找替代方案]
```

**分析维度：**

1. **版本号分析**：对比 npm registry 中版本号与 Vue3 的兼容性
2. **社区活跃度**：检查库的维护状态（github 最近更新、issue 回复等）
3. **官方声明**：检查官方文档是否声明支持 Vue3
4. **替代方案**：对不支持 Vue3 的库寻找替代方案

### 4.3 报告生成阶段

```
[生成依赖升级报告] → [生成依赖安装脚本] → [提交给迁移总指挥]
```

## 六、特殊依赖处理指南

### 6.1 Vue.prototype 对应的全局 API
在 Vue2 中通过 `Vue.prototype.xxx = yyy` 注入的全局 API，在 Vue3 中需要通过 `app.config.globalProperties.xxx = yyy` 实现。

**依赖分析专家**需要特别记录：
- 全局 API 的名称和类型
- 它们被使用到的组件范围
- 推荐的在 Vue3 中的实现方式

### 6.2 Vue2 过滤器 (filters)
Vue3 移除了过滤器功能，需要：
- 将过滤器转换为全局方法或组件方法
- 或使用计算属性替代
- 或使用组合式函数 (composables) 替代

### 6.3 Vue2 混入 (mixins)
Vue3 依然支持混入，但推荐使用组合式函数 (Composables) 替代：
- 简单的 mixin 可以直接迁移
- 复杂的 mixin 建议重构为 composables

### 6.4 Vue2 插件 (Vue.use)
Vue2 插件通过 `Vue.use()` 注册，Vue3 通过 `app.use()` 注册：
- 分析所有 `Vue.use()` 注册的插件
- 检查插件是否支持 Vue3
- 如不支持则寻找替代方案

### 6.5 自定义指令
Vue3 中自定义指令的钩子函数名发生了变化：
- `bind` → `beforeMount`
- `inserted` → `mounted`
- `update` → `updated`
- `componentUpdated` → `beforeUpdate`
- `unbind` → `unmounted`

## 七、外部 Script 标签处理策略

| 情况 | 处理方式 | 示例 |
|------|----------|------|
| 兼容 Vue3 的 CDN 资源 | 保留并升级版本 | CDN 上的 Vue 3 版本 |
| 不兼容 Vue3 的 CDN 资源 | 替换为兼容版本或 npm 包 | 某些旧版统计脚本 |
| 项目内部脚本 | 迁移到 Vue3 项目中 | 内部工具库 |
| 已弃用的外部资源 | 移除 | 不再使用的 CDN 资源 |
| 需要 npm 安装的 | 改为 npm 包引入 | 统一包管理 |

## 九、依赖安装脚本生成

依赖分析专家会为架构专家生成可以直接执行的依赖安装命令：

```bash
# ===== 运行时依赖安装 =====
# 核心框架升级
npm install vue@^3.4.0

# 路由升级
npm install vue-router@^4.3.0

# 状态管理（Vuex → Pinia）
npm install pinia@^2.1.0

# UI 组件库升级
npm install element-plus@^2.5.0

# 其他依赖
npm install axios@^1.6.0

# ===== 开发依赖安装 =====
npm install -D vite@^5.0.0
npm install -D @vitejs/plugin-vue@^5.0.0
npm install -D typescript@^5.3.0
npm install -D vue-tsc@^2.0.0

# ===== 移除不兼容依赖 =====
npm uninstall vue-template-compiler
npm uninstall vue-loader
npm uninstall vuex
```

## 十、注意事项

1. **版本锁定**：明确记录每个依赖的目标版本号，避免依赖版本不统一导致的问题
2. **副作用检测**：注意某些依赖可能在安装时有副作用（如 postinstall 脚本），需要评估影响
3. **TypeScript 类型支持**：检查依赖是否提供 TypeScript 类型定义，如不提供需要创建自定义类型声明
4. **CSS 框架兼容性**：注意 Less/Sass/PostCSS 等样式工具的版本兼容性
5. **Node.js 版本要求**：检查 Vue3 及相关依赖对 Node.js 版本的要求，确保开发环境满足要求
6. **CDN 转 npm**：建议将 CDN 引入的外部资源转为 npm 包管理，确保版本可控
7. **渐进增强**：如某些依赖确实无法升级或找到替代方案，需要标记并请人工介入评估
8. **与架构专家的配合**：依赖分析完成后将结果交付给架构专家安装，同时将升级对照表交付给迁移专家参考
