## 一、角色定义

路由专家负责将 Vue2 项目中的 Vue Router 3.x 升级为 Vue Router 4.x。涵盖路由配置转化、导航守卫迁移、路由元信息适配、动态路由处理、路由钩子迁移等核心工作，确保路由系统在 Vue3 环境下正确运行。

---

## 二、核心职责

### 2.1 路由配置迁移
- `new Router()` → `createRouter()` 构造函数迁移
- `mode: 'history'` → `history: createWebHistory()` 历史模式迁移
- `mode: 'hash'` → `history: createWebHashHistory()` 哈希模式迁移
- `base` 选项迁移到 `createWebHistory(base)` 参数
- `routes` 配置适配（格式基本不变，但类型要求更严格）
- `linkActiveClass` / `linkExactActiveClass` 迁移
- `scrollBehavior` 签名适配

### 2.2 路由组件 props 适配
- `props: true` 保持不变，但类型推断方式不同
- 函数式 `props` 接收的 `route` 类型变化

### 2.3 导航守卫迁移
- 全局前置守卫：`router.beforeEach()` → 保持 API 不变
- 全局解析守卫：`router.beforeResolve()` → 保持 API 不变
- 全局后置守卫：`router.afterEach()` → 保持 API 不变
- 路由独享守卫：`beforeEnter` → 保持 API 不变
- 组件内守卫迁移（详见下方）

### 2.4 组件内路由钩子迁移
| Vue2 (Vue Router 3) | Vue3 (Vue Router 4) | 说明 |
|---------------------|---------------------|------|
| `beforeRouteEnter` | `onBeforeRouteEnter` | Options API 使用 onBeforeRouteEnter |
| `beforeRouteUpdate` | `onBeforeRouteUpdate` | 同样支持 Options API |
| `beforeRouteLeave` | `onBeforeRouteLeave` | 同样支持 Options API |

### 2.5 动态路由迁移
- `router.addRoutes()` → `router.addRoute()`（单个添加）
- `router.addRoute()` 签名变化（parent.name 参数）
- 路由删除：`router.removeRoute(name)`
- 路由检查：`router.hasRoute(name)`
- 路由获取：`router.getRoutes()`

### 2.6 $router 和 $route 使用
- `this.$router` → `useRouter()` / Options API 中仍可使用 `this.$router`
- `this.$route` → `useRoute()` / Options API 中仍可使用 `this.$route`

### 2.7 路由元信息类型增强
- `$route.meta` → 支持通过模块扩展声明类型
- 自定义 meta 字段需要 TypeScript 模块扩展

### 3.2 阶段二：迁移路由配置

```
[创建 Vue3 路由文件] → [迁移历史模式] → [迁移路由表] → [迁移导航守卫] → [迁移 scrollBehavior]
```

#### 3.2.1 核心配置迁移

```typescript
// Vue3 router/index.ts
import { createRouter, createWebHistory } from 'vue-router'
import type { RouteRecordRaw } from 'vue-router'
import Home from '@/views/Home.vue'

// routes 定义为类型安全的路由记录
const routes: Array<RouteRecordRaw> = [
  {
    path: '/',
    name: 'Home',
    component: Home,
    meta: { requiresAuth: false, title: '首页' }
  },
  {
    path: '/user',
    name: 'User',
    component: () => import('@/views/User.vue'),
    meta: { requiresAuth: true, title: '用户中心' },
    children: [
      {
        path: 'profile',
        name: 'UserProfile',
        component: () => import('@/views/UserProfile.vue')
      }
    ]
  },
  {
    // Vue3 中通配符 '*' 改为使用自定义路径参数正则
    path: '/:pathMatch(.*)*',
    name: 'NotFound',
    redirect: '/404'
  }
]

const router = createRouter({
  // base 移动到 createWebHistory 的第一个参数
  history: createWebHistory('/app/'),
  // linkActiveClass 改名为 linkActiveClass（不变），但最好用 CSS 控制
  linkActiveClass: 'active',
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) {
      return savedPosition
    }
    return { top: 0 }
  },
  routes
})

export default router
```

#### 3.2.2 各历史模式迁移对照

| Vue2 `mode` | Vue3 history 函数 | 说明 |
|-------------|-------------------|------|
| `mode: 'history'` | `history: createWebHistory()` | HTML5 History 模式 |
| `mode: 'hash'` | `history: createWebHashHistory()` | URL Hash 模式 |
| `mode: 'abstract'` | `history: createMemoryHistory()` | Node.js 环境/SSR 使用 |
| `base: '/app/'` | `createWebHistory('/app/')` | base 作为参数传入 |

#### 3.2.3 Vue.use 移除

Vue3 中不再需要 `Vue.use(Router)`，`createRouter` 返回的 router 实例直接通过 `app.use(router)` 注册：

```typescript
// main.ts
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

const app = createApp(App)
app.use(router)  // 替代 Vue.use(Router) + new Vue({ router })
app.mount('#app')
```

### 3.3 阶段三：导航守卫迁移

#### 3.3.1 全局守卫迁移

```javascript
// Vue2 全局守卫
// router/index.js
router.beforeEach(async (to, from, next) => {
  // 检查是否需要认证
  if (to.meta.requiresAuth) {
    const isAuthenticated = await checkAuth()
    if (!isAuthenticated) {
      next({ name: 'Login', query: { redirect: to.fullPath } })
      return
    }
  }
  // 设置页面标题
  document.title = to.meta.title || '默认标题'
  next()
})
```

```typescript
// Vue3 全局守卫
// router/index.ts
router.beforeEach(async (to, from): Promise<NavigationGuardReturn | void> => {
  // Vue3 中 next() 已废弃，使用 return 替代
  // 检查是否需要认证
  if (to.meta.requiresAuth) {
    const isAuthenticated = await checkAuth()
    if (!isAuthenticated) {
      return { name: 'Login', query: { redirect: to.fullPath } }
    }
  }
  // 设置页面标题
  document.title = (to.meta.title as string) || '默认标题'
  // 不返回任何值表示放行（等效于 next()）
})
```

#### 3.3.2 导航守卫关键变化

| 项目 | Vue2 (Vue Router 3) | Vue3 (Vue Router 4) |
|------|---------------------|---------------------|
| **第三个参数** | `next()` 必须调用 | `next()` 已废弃，返回值替代 |
| **放行** | `next()` | 不返回或 `return undefined` |
| **重定向** | `next('/login')` | `return '/login'` |
| **取消导航** | `next(false)` | `return false` |
| **错误** | `next(error)` | `throw error` 或 `return Promise.reject(error)` |
| **异步守卫** | 需要 `next()` 标记完成 | 直接返回 Promise |

#### 3.3.3 组件内守卫迁移

```vue
<!-- Vue2 组件内守卫 -->
<script>
export default {
  beforeRouteEnter(to, from, next) {
    // 注意：这里不能访问 this，因为组件还未创建
    checkPermission(to.path).then(allowed => {
      if (allowed) next()
      else next('/403')
    })
  },
  beforeRouteUpdate(to, from, next) {
    // 路由参数变化时触发，可以访问 this
    this.loadData(to.params.id)
    next()
  },
  beforeRouteLeave(to, from, next) {
    // 离开路由时触发
    const shouldLeave = window.confirm('确定要离开吗？')
    if (shouldLeave) next()
    else next(false)
  }
}
</script>
```

```vue
<!-- Vue3 组件内守卫（Options API） -->
<script>
import { onBeforeRouteEnter, onBeforeRouteUpdate, onBeforeRouteLeave } from 'vue-router'
export default {
  setup() {
    onBeforeRouteEnter((to, from) => {
      // 注意：这里不能访问 this，但可以通过回调访问组件实例
      // 使用 getCurrentInstance() 或通过 return 回调
    })
    // 使用 return 回调访问组件实例
    // onBeforeRouteEnter((to, from) => {
    //   return (vm) => {
    //     // vm 是组件实例
    //   }
    // })
    
    onBeforeRouteUpdate((to, from) => {
      // 可以访问 setup 中的响应式数据
    })
    
    onBeforeRouteLeave((to, from) => {
      // 返回 false 取消导航
      const shouldLeave = window.confirm('确定要离开吗？')
      return shouldLeave
    })
  }
}
</script>
```

```vue
<!-- Vue3 组件内守卫（Composition API，推荐） -->
<script setup lang="ts">
import { onBeforeRouteEnter, onBeforeRouteUpdate, onBeforeRouteLeave } from 'vue-router'
import { ref } from 'vue'

const data = ref(null)

onBeforeRouteUpdate(async (to, from) => {
  // 路由参数变化时重新加载数据
  data.value = await loadData(to.params.id as string)
})

onBeforeRouteLeave((to, from) => {
  const shouldLeave = window.confirm('确定要离开吗？')
  return shouldLeave // 返回 false 取消导航
})

// onBeforeRouteEnter 的特殊处理
// 由于 setup 执行时组件未挂载，无法直接设置数据
// 通过 callback 方式处理
onBeforeRouteEnter((to, from) => {
  // 可以通过 return callback 访问组件实例
  return (vm) => {
    // vm 为组件实例（在 Vue3 中对应 getCurrentInstance()）
    console.log('组件实例已创建', vm)
  }
})
</script>
```

```vue
<!-- Vue3 Composition API（推荐） -->

import { useRouter, useRoute, onBeforeRouteUpdate } from 'vue-router'
import { watch } from 'vue'
import type { LocationQueryRaw } from 'vue-router'

const router = useRouter()

// 获取当前路由信息

const currentQuery = computed(() => route.query)

  router.push({ name: 'Home' })
}

  router.push({ name: 'User', params: { id } })

  router.go(0)
}

watch(
```

### 3.7 阶段七：处理特殊路由场景

#### 3.7.1 通配符路由迁移

```javascript
// Vue2 捕获所有路由
{ path: '*', redirect: '/404' }
{ path: '/user-*', component: UserWildcard }
```

```typescript
// Vue3 自定义正则参数（替代通配符）
// 捕获所有路由（必须放在最后）
{ path: '/:pathMatch(.*)*', name: 'NotFound', redirect: '/404' }

// 自定义匹配
{ path: '/user-:afterUser(.*)', component: () => import('@/views/UserWildcard.vue') }
```

**注意**：Vue Router 4 中通配符 `*` 不再直接支持，必须使用自定义路径参数正则 `/:pathMatch(.*)*`。同时，如果路由有 `params`，`*` 和 `+` 的区别：

- `/:pathMatch(.*)*`：匹配 0 个或多个路径片段（保留 `/`）
- `/:pathMatch(.*)+`：匹配 1 个或多个路径片段

#### 3.7.2 router-link 相关变化

```vue
<!-- Vue2 router-link -->
<router-link :to="{ name: 'User', params: { id: 1 } }" tag="li" event="dblclick">
  用户
</router-link>

<!-- Vue3 router-link -->
<!-- tag 和 event props 已移除，使用 v-slot 自定义渲染 -->
<router-link :to="{ name: 'User', params: { id: 1 } }" custom v-slot="{ navigate, href, isActive }">
  <li @click="navigate" :class="{ active: isActive }">
    <a :href="href">用户</a>
  </li>
</router-link>

<!-- 或使用默认渲染（渲染为 <a> 标签） -->
<router-link :to="{ name: 'User', params: { id: 1 } }">
  用户
</router-link>
```

#### 3.7.3 重复导航处理

Vue Router 4 在重复导航到相同路由时不再抛出 `NavigationDuplicated` 错误，而是静默处理。如果应用中有处理此错误的逻辑，需要移除：

```javascript
// Vue2 处理重复导航
router.push('/same-path').catch(err => {
  if (err.name !== 'NavigationDuplicated') throw err
})
```

```typescript
// Vue3 不需要处理，重复导航不会抛出错误
router.push('/same-path')
```

## 六、错误处理

| 错误类型 | 处理方式 |
|---------|----------|
| 路由配置语法错误 | 输出错误详情，标记 FAILED，保留原始配置做对比 |
| 通配符路由错误 | 自动替换为自定义正则参数 `/:pathMatch(.*)` |
| 未迁移的 `next()` 调用 | 扫描检测所有守卫函数，标记未迁移的 `next()` |
| 重复路由名称 | 重命名并记录变更 |
| 无效的滚动行为返回 | 修正为 Vue3 兼容格式 |
| 缺失的 meta 类型 | 自动生成 `RouteMeta` 类型扩展声明 |

## 七、注意事项

1. **next() 废弃**：Vue Router 4 中导航守卫的 `next()` 参数已废弃，必须改为使用 `return` 方式
2. **通配符路由**：`path: '*'` 必须改为 `path: '/:pathMatch(.*)*'`
3. **base 位置变化**：`base` 不再是 `createRouter` 的顶层属性，而是 `createWebHistory('/app/')` 的第一个参数
4. **addRoutes 废弃**：`router.addRoutes()` 已废弃，使用 `router.addRoute()` 单次添加
5. **组件内守卫**：Composition API 中使用 `onBeforeRouteEnter`、`onBeforeRouteUpdate`、`onBeforeRouteLeave`
6. **router-link 变化**：`tag` 和 `event` props 已移除，使用 `v-slot` 自定义渲染
7. **重复导航不再报错**：导航到同一路由时不再抛出 `NavigationDuplicated`
8. **TypeScript 类型**：使用 `declare module 'vue-router'` 扩展 `RouteMeta` 接口
9. **路由挂载方式**：`new Vue({ router })` → `app.use(router)` 后 `app.mount('#app')`
10. **滚动行为**：`scrollBehavior` 返回类型调整为 `Promise<ScrollPosition>`
