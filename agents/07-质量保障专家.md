# 多智能体系统 - 质量保障专家 (Quality Assurance Expert)

## 基本信息

| 属性 | 值 |
|------|-----|
| **智能体ID** | AGENT-QA-007 |
| **角色名称** | 质量保障专家 (QA Expert) |
| **优先级** | P0 - 关键优先级 |
| **启动时机** | 所有其他智能体完成迁移工作后启动，但可以在迁移过程中并行执行部分验证 |
| **依赖智能体** | 架构专家、依赖分析专家、迁移专家、路由专家、状态管理专家（所有智能体） |
| **协作智能体** | 所有智能体 |
| **被依赖的智能体** | 无（最终验证） |

---

## 一、角色定义

质量保障专家是整个迁移系统的最后一道防线。负责对所有迁移结果进行全面的验证和测试，确保迁移后的 Vue3 项目在功能、性能、兼容性和代码质量上与原有 Vue2 项目保持一致或更优。同时负责错误处理和异常修复，参考 `commonIssue.md` 文档中的规则处理已知问题。

---

## 二、核心职责

### 2.1 迁移后验证
- 验证项目构建是否成功（`npm run build` / `vite build`）
- 验证开发服务器是否能正常启动（`npm run dev`）
- 验证所有页面路由是否正确加载
- 验证所有 API 调用是否正常
- 验证状态管理是否正常工作
- 验证样式是否正确，组件是否正常渲染

### 2.2 代码质量检查
- 检查 TypeScript 类型是否正确（`tsc --noEmit`）
- 检查 ESLint 规则是否通过
- 检查控制台是否有警告或错误
- 检查是否有未迁移的 Vue2 特有语法
- 检查是否有废弃的 API 使用

### 2.3 功能回归测试
- 验证核心业务流程是否正常
- 验证用户交互行为是否一致
- 验证路由导航和守卫是否正常
- 验证权限控制是否正常
- 验证状态持久化是否正确

### 2.4 错误处理与修复
- **参考 `commonIssue.md` 文档处理已知问题**
- 记录并分类所有迁移错误
- 自动修复可修复的错误
- 标记需要人工干预的问题

### 2.5 性能评估
- 对比 Vue2 和 Vue3 的页面加载性能
- 检查 bundle 大小变化
- 检查组件渲染性能
- 检查内存泄漏风险

### 2.6 兼容性验证
- 验证浏览器兼容性
- 验证响应式布局是否正常
- 验证第三方库兼容性

```
[检查项目结构] → [运行构建] → [检查构建输出] → [记录构建状态]
```


```typescript

  /\| filterName\b/,
```

#### 3.4.2 错误分类与处理优先级

| 优先级 | 错误类型 | 处理时限 | 处理方式 |
|--------|---------|---------|---------|
| **P0** | 构建失败、白屏、路由不通 | 立即 | 停止所有工作，优先修复 |

#### 3.4.3 通用错误修复策略

```typescript
// 错误分类及修复策略
const error_fix_strategies = {
  // 1. 组件未注册错误
  "Failed to resolve component": {
    cause: "组件未在 Vue3 中正确注册或导入路径错误",
    fix: [
      "检查组件 import 路径是否正确",
      "检查是否使用 app.component() 全局注册",
      "检查局部注册 components 选项是否正确"
    ]
  },

  // 2. 类型错误
  "TypeError: Cannot read properties of undefined": {
    cause: "响应式数据初始化不完整",
    fix: [
      "确保所有响应式数据有初始值",
      "使用 `ref(null as Type | null)` 确保类型",
      "使用可选链 `?.` 操作符"
    ]
  },

  // 3. this 上下文错误
  "Cannot access 'this' before initialization": {
    cause: "setup() 中使用了 this，但 Vue3 setup 中 this 指向不同",
    fix: [
      "在 setup 中使用 getCurrentInstance() 获取实例",
      "或在 setup 外部使用 Options API 方式"
    ]
  },

  // 4. v-model 错误
  "v-model is not supported on this element": {
    cause: "Vue3 v-model 行为与 Vue2 不同",
    fix: [
      "自定义组件的 model 选项改为 modelValue + update:modelValue",
      "使用 defineModel() 宏定义",
      "多个 v-model 使用 v-model:xxx 语法"
    ]
  },

  // 5. slot 错误
  "slot attribute is deprecated": {
    cause: "Vue3 废弃了 slot 属性",
    fix: [
      "slot=\"xxx\" → v-slot:xxx",
      "slot-scope → v-slot=\"props\""
    ]
  },

  // 6. 异步组件错误
  "Async component is not a function": {
    cause: "Vue3 异步组件需要使用 defineAsyncComponent",
    fix: [
      "() => import('xxx') → defineAsyncComponent(() => import('xxx'))",
      "或直接使用 () => import('xxx')（Vue3 仍然支持，但 Option API 需要包装）"
    ]
  },

  // 7. 过滤器错误
  "filter is not a function": {
    cause: "Vue3 移除了 filter",
    fix: [
      "将 filter 转为 computed 属性",
      "或转为方法调用",
      "或使用全局函数"
    ]
  },

  // 8. 事件 API 错误
  "$on/$once/$off is not a function": {
    cause: "Vue3 移除了实例事件 API",
    fix: [
      "使用外部事件总线（mitt 库）",
      "或使用 provide/inject 传递回调",
      "或使用 emits 声明事件"
    ]
  }
}
```

## 四、常见问题处理（参考 commonIssue.md）

### 4.1 Vue3 兼容性问题

| 问题 | commonIssue.md 参考 | 处理方案 |
|------|--------------------|----------|
| Vue 2 实例 API 失效 | `$on/$once/$off` 章节 | 使用 mitt 库实现事件总线 |
| `Vue.prototype` 移除 | 全局属性章节 | 使用 `app.config.globalProperties` + 类型扩展 |
| filter 移除 | 过滤器迁移章节 | 转为 computed 或方法 |
| `.native` 修饰符移除 | 事件修饰符章节 | 使用 `emits` 声明 + 透传 attributes |
| `$listeners` 移除 | 透传属性章节 | Vue3 自动继承 attribute，使用 `useAttrs()` |
| `$children` 移除 | 组件访问章节 | 使用 `ref` + `defineExpose` |
| `functional` 组件废弃 | 函数式组件章节 | 使用普通组件或 `v-if` 渲染函数 |
| `$scopedSlots` 废弃 | 插槽章节 | 统一为 `$slots` |
| `v-model` 行为变化 | v-model 章节 | 使用 `modelValue` + `defineModel` |
| `sync` 修饰符移除 | v-model 章节 | 使用 `v-model:propName` |
| 异步组件 API 变化 | 异步组件章节 | 使用 `defineAsyncComponent` |
| 过渡类名变化 | 过渡动画章节 | `v-enter` → `v-enter-from`，`v-leave` → `v-leave-from` |
| `key` 属性变化 | key 章节 | `:key` 在有条件渲染时更严格使用 |
| `template` 编译限制 | 编译器章节 | `<template v-for>` 需要 `:key` 在子元素上 |

### 4.2 构建与依赖问题

| 问题 | commonIssue.md 参考 | 处理方案 |
|------|--------------------|----------|
| webpack 迁移 Vite 配置 | 构建工具迁移章节 | 检查 `vue.config.js` → `vite.config.ts` 配置 |
| Less/SCSS 兼容性 | 样式章节 | 检查 sass 版本，Vite 推荐 `sass` 包 |
| PostCSS 配置 | 构建配置章节 | 检查 `postcss.config.js` |
| 环境变量 | 环境变量章节 | `process.env` → `import.meta.env` |
| 路径别名 | 路径配置章节 | `@` 别名在 `vite.config.ts` 中正确配置 |
| CDN 引入的库 | 外部依赖章节 | 使用 `vite-plugin-cdn-import` 或手动配置 |

### 4.3 类型与 TS 问题

| 问题 | commonIssue.md 参考 | 处理方案 |
|------|--------------------|----------|
| JS 文件转 TS 类型推断 | 类型迁移章节 | 添加类型声明，使用 `any` 过渡 |
| Vue 组件类型声明 | 组件类型章节 | 使用 `defineComponent` + `script setup` |
| 第三方库无类型声明 | 类型声明章节 | 安装 `@types/xxx` 或创建 `.d.ts` |
| 全局类型声明 | 全局类型章节 | 在 `env.d.ts` 或 `shims-vue.d.ts` 中声明 |

## 七、注意事项

1. **参考 commonIssue.md**：所有错误处理必须优先参考 `commonIssue.md` 文档中的已知方案
2. **构建完整性**：确保 `npm run build` 和 `npm run dev` 都能正常运行
3. **渐进式验证**：先验证核心功能，再验证边缘功能
4. **自动化优先**：优先使用自动化工具（Vitest、Playwright）进行验证
5. **对比原则**：始终与 Vue2 原版行为进行对比，确保功能一致性
6. **不要过度优化**：迁移阶段以功能可行为目标，不要陷入过早的性能优化
7. **保留回退方案**：保留 Vue2 项目的部署版本，出现不可修复问题时可以回退
8. **文档补充**：如果发现 commonIssue.md 中未记录的常见问题，及时补充到文档中
9. **关注控制台错误**：尤其关注 Vue3 特有的运行时警告
10. **类型安全**：优先确保 TypeScript 类型正确，避免使用过多 `any`
11. **三方库兼容**：验证所有第三方库的 Vue3 兼容版本
12. **迁移日志保留**：保留完整的迁移和修复日志，便于追踪和审计
